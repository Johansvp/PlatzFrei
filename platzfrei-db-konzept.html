Konzeptioneller Datenbankentwurf – Strukturiert nach App-Flow

Die Tabellen sind in der Reihenfolge dargestellt, in der sie bei der Nutzung der PlatzFrei-App zum Einsatz kommen:
① Anmeldung → ② Home / Suche → ③ Platz buchen → ④ Buchungsbestätigung → ⑤ Freunde einladen → ⑥ Profil, Gruppen & Chat
PK = Primärschlüssel  ·  FK = Fremdschlüssel  ·  Beziehungen sind als Kardinalitäten angegeben (1:N, M:N).

1
Anmeldung
Login / Registrierung
Der Nutzer meldet sich an oder registriert sich neu. Die User-Tabelle ist die zentrale Entität der gesamten App – alle anderen Tabellen verweisen auf sie.
User
Alle App-Nutzer
PKuser_idINT
—vornameVARCHAR(50)
—nachnameVARCHAR(50)
—emailVARCHAR · UNIQUE
—passwort_hashVARCHAR(255)
—profilbild_urlTEXT
—erstellt_amDATETIME
2
Home
Plätze entdecken & filtern
Auf dem Homescreen sieht der Nutzer verfügbare Sportanlagen. Die App lädt Sportanlagen, filtert nach Kategorie (Sportart) und zeigt die einzelnen Plätze je Anlage an.
Sportanlage
Buchbare Anlage
PKanlage_idINT
—nameVARCHAR(100)
—adresseTEXT
—latitudeFLOAT
—longitudeFLOAT
—bewertungFLOAT
—bild_urlTEXT
Kategorie
Sportart je Anlage
PKkategorie_idINT
FKanlage_idINT
—sportartVARCHAR(50)
—icon_urlTEXT
Platz
Einzelner Sportplatz
PKplatz_idINT
FKanlage_idINT
—bezeichnungVARCHAR(50)
—kapazitaetINT
—sport_typVARCHAR(50)
Sportanlage→Kategorie 1:N — eine Anlage hat mehrere Sportarten
Sportanlage→Platz 1:N — eine Anlage hat mehrere Plätze
3
Platz buchen
Datum & Zeitslot wählen
Der Nutzer wählt einen Platz, ein Datum und eine Uhrzeit. Ein neuer Datensatz wird in der Buchung-Tabelle angelegt und mit User und Platz verknüpft.
Buchung
Platzreservierung
PKbuchung_idINT
FKuser_idINT→ User
FKplatz_idINT→ Platz
—datumDATE
—startzeitTIME
—endzeitTIME
—statusENUMausstehend · bestätigt · storniert
—erstellt_amDATETIME
User→Buchung 1:N — ein Nutzer erstellt viele Buchungen
Platz→Buchung 1:N — ein Platz wird mehrfach (zu versch. Zeiten) gebucht
4
Bestätigung
Automatische Benachrichtigung
Nach der Buchung wird der Status in Buchung auf „bestätigt" gesetzt. Gleichzeitig erzeugt das System automatisch einen Eintrag in Benachrichtigung — die Buchungsbestätigung geht an den Nutzer (und später an eingeladene Freunde).
Buchung
Status-Update: bestätigt
PKbuchung_idINT
↑status→ 'bestätigt'
Benachrichtigung
Automatische Mitteilung
PKbenachrichtigung_idINT
FKempfaenger_idINT→ User
FKbuchung_idINT→ Buchung
FKeinladung_idINT · NULL→ Einladung
—typENUMbuchungsbestaetigung · einladung · erinnerung · stornierung
—inhaltTEXT
—gesendet_amDATETIME
—gelesenBOOLEAN
Buchung→Benachrichtigung 1:N — eine Buchung löst mehrere Benachrichtigungen aus
User→Benachrichtigung 1:N — ein Nutzer empfängt viele Benachrichtigungen
5
Freunde einladen
Einladung & Benachrichtigung
Nach der Bestätigung kann der Bucher Freunde einladen. Für jeden eingeladenen Freund wird ein Eintrag in Einladung angelegt. Das System schreibt automatisch eine Benachrichtigung an den eingeladenen Freund. Die Freundesliste wird aus Freundschaft geladen.
Einladung
Freunde zur Buchung einladen
PKeinladung_idINT
FKbuchung_idINT→ Buchung
FKeinlader_idINT→ User
FKeingeladener_idINT→ User
—statusENUMausstehend · angenommen · abgelehnt
—erstellt_amDATETIME
Freundschaft
M:N User ↔ User
PKfreundschaft_idINT
FKuser_id_1INT→ User
FKuser_id_2INT→ User
—statusENUMausstehend · angenommen
—erstellt_amDATETIME
Buchung→Einladung 1:N — pro Buchung mehrere Einladungen möglich
Einladung→Benachrichtigung 1:N — jede Einladung erzeugt automatisch eine Benachrichtigung
User↔User M:N — Freundschaftsbeziehungen (Zwischentabelle: Freundschaft)
6
Profil
Gruppen & Chat
Im Profil sieht der Nutzer seine Buchungshistorie, Gruppen und Freunde. Gruppe und GruppeMitglied verwalten Teams. Nachricht speichert den Gruppen- und Direktchat.
